"MCPwn" — The Nginx Exploit Named After the Protocol It Betrays

The Hacker News · Picus Security · April 15, 2026

The irony is almost too perfect: a critical authentication bypass vulnerability in nginx-ui has been nicknamed MCPwn by researchers at Pluto Security because it exploits nginx-ui's support for Model Context Protocol servers — the very protocol that Anthropic spent two years championing as the universal interface for AI systems. CVE-2026-33032 carries a CVSS score of 9.8 and enables full takeover of the web server in two HTTP requests.

Here's how it works: nginx-ui added MCP server support in late 2025, exposing two HTTP-accessible endpoints for AI model communication. One of them — /mcp_message — was implemented without any authentication, giving any host on the network access to 12 MCP tools including config writes with automatic nginx reload. An attacker sends a GET to /mcp to get a session ID, then a POST to /mcp_message to invoke any tool. Full takeover in seconds.

About 2,689 instances are exposed on the internet. Recorded Future has listed it among the 31 most actively exploited vulnerabilities in March 2026. Patch immediately if you run nginx-ui.
